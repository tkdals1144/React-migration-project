## 📝 프론트엔드/백엔드 인터페이스 규약 (프로젝트 시작 필수 사양)

본 문서는 프로젝트 시작 전, 프론트엔드와 백엔드 간의 효율적인 협업과 데이터 일관성을 보장하기 위해 정의하는 핵심 규약입니다.

---

### # 1. API 인터페이스 규약 (API Interface Contract)

프론트엔드와 백엔드가 데이터를 주고받는 방식에 대한 가장 기본적인 계약입니다.

| 사양 항목 | 설명 | 예시 및 세부 규약 |
| :--- | :--- | :--- |
| **URL 엔드포인트** | 리소스 기반의 명확한 주소 체계를 사용하며, URL 버전 관리를 포함할 수 있음. | `GET /api/v1/users` (전체 사용자 조회), `POST /api/v1/users/login` |
| **HTTP 메서드** | RESTful 원칙에 따라 HTTP 메서드를 용도에 맞게 정의. | **조회:** `GET`, **생성:** `POST`, **수정(전체):** `PUT`, **수정(부분):** `PATCH`, **삭제:** `DELETE` |
| **요청/응답 형식** | 통신 데이터 형식은 JSON으로 통일하며, 필드 이름과 데이터 타입을 확정. | **Content-Type:** `application/json` 사용을 기본으로 함. |
| **상태 코드 규약** | 모든 API 응답은 표준 HTTP 상태 코드를 반환하며, 프론트엔드는 이에 따라 로직을 처리. | **200 OK:** 성공, **201 Created:** 생성 성공, **400 Bad Request:** 유효성 검사 실패, **401 Unauthorized:** 인증 오류, **500 Internal Server Error:** 서버 오류 |
| **인증/인가 방식** | 사용자 인증 및 권한 확인을 위한 토큰 전달 방식 정의. | **JWT (JSON Web Token)** 사용. 토큰은 **HTTP Header의 `Authorization: Bearer <token>`** 형태로 전달. |

### # 2. 에러 핸들링 규약 (Error Handling Contract)

백엔드에서 발생한 오류를 프론트엔드가 일관되고 사용자 친화적으로 처리할 수 있도록 에러 응답 형식을 통일합니다.

| 사양 항목 | 설명 | 예시 응답 JSON 형식 |
| :--- | :--- | :--- |
| **공통 에러 형식** | `4xx` 또는 `5xx` 상태 코드가 반환될 때, 응답 본문은 항상 이 형식을 따름. | ```json { "code": "E001", "message": "유효하지 않은 요청 데이터입니다.", "details": "필드 'username'은 필수입니다." } ``` |
| **에러 코드 (Code)** | 백엔드에서 정의한 고유한 에러 식별자 (프론트엔드가 이를 바탕으로 메시지 번역). | **`E`** (일반 에러), **`A`** (인증/권한 에러), **`B`** (비즈니스 로직 에러) 등의 접두사 사용. |
| **메시지 (Message)** | 개발자가 참고할 수 있는 상세한 설명 (사용자에게 직접 노출되지 않도록 주의). | 한국어 또는 영어로 작성된 오류 상황 설명. |

### # 3. 데이터 및 네이밍 컨벤션

API 인터페이스에서 사용되는 필드 이름에 대한 명명 규칙을 정의합니다.

| 항목 | 규약 내용 | 예시 |
| :--- | :--- | :--- |
| **API 필드 이름 표기법** | API 요청/응답 본문 내의 모든 필드 이름은 `camelCase`로 통일합니다. (백엔드 내부의 데이터베이스나 코드는 별도 규칙을 따를 수 있으나, API 통신 시에는 변환하여 제공). | `userName`, `postId`, `totalCount` |
| **날짜/시간 형식** | 모든 날짜 및 시간 데이터는 ISO 8601 형식의 문자열로 전송합니다. | `2025-12-15T10:30:00Z` |
| **Boolean 값** | 상태를 나타내는 값은 문자열이 아닌 **JSON의 불리언(Boolean) 타입** (`true` 또는 `false`)을 사용합니다. | `{"isLoggedIn": true}` |

### # 4. 기타 환경 및 협업 사양

| 사양 항목 | 내용 |
| :--- | :--- |
| **개발 환경 설정** | 로컬 개발 서버의 주소 및 포트를 정의하여 동기화합니다. | **프론트엔드 URL:** `http://localhost:3000`, **백엔드 API URL:** `http://localhost:8080/api/v1` |
| **Git 컨벤션** | 브랜치 이름, 커밋 메시지 규칙 등을 정의하여 협업 시 이력 관리를 용이하게 합니다. | **브랜치 명:** `feat/feature-name`, `fix/bug-description`. **커밋 메시지:** Conventional Commits 규약 준수. |
